### 1. Brak implementacji CallDepthGuard / SessionManager / ArchitectureValidator
- **Lokalizacja:** `_projects/registry/service_registry.py`
- **Problem:** `master_prompt_ligo_2.0.md` i `project_anchor.md` opisują CallDepthGuard (blokada rekurencji, max_depth=20), SessionManager (JSON snapshoty stanu) oraz ArchitectureValidator (walidacja cross-module imports). Jeśli te komponenty nie istnieją w kodzie, dokumentacja v2.0 jest nieprawdziwa.
- **Ryzyko:** Fałszywe poczucie bezpieczeństwa — kod może zawierać infinite loops bez wykrycia.
- **Poprawka:** Zaimplementować brakujące komponenty lub zaktualizować dokumentację do stanu rzeczywistego.

```python
# Przykład CallDepthGuard do wdrożenia:
import threading

_call_depth = threading.local()

class CallDepthGuard:
    """Blokuje rekurencję i infinite loops."""
    MAX_DEPTH = 20
    
    @classmethod
    def enter(cls, context: str = "") -> int:
        _call_depth.value = getattr(_call_depth, 'value', 0) + 1
        if _call_depth.value > cls.MAX_DEPTH:
            raise RecursionError(
                f"LOOP_DETECTED: Call depth {_call_depth.value} exceeds max {cls.MAX_DEPTH}. "
                f"Context: {context}"
            )
        return _call_depth.value
    
    @classmethod
    def exit(cls) -> None:
        _call_depth.value = max(0, getattr(_call_depth, 'value', 1) - 1)
```

### 4. `sys.path.insert(0, ...)` w runtime — niebezpieczna praktyka
- **Lokalizacja:** `_projects/orchestrator/main.py:20-22` oraz `_projects/tests/test_mvg.py:14`
- **Problem:** Ręczne modyfikowanie `sys.path` w runtime łamie konwencje pakietowania Pythona. Może powodować konflikty nazw, nieprzewidywalne zachowanie w różnych środowiskach.

include = ["_projects*", "_system*"]

### 5. `bootstrap()` łamie zasadę OCP (Open/Closed Principle)
- **Lokalizacja:** `_projects/orchestrator/main.py:35-73`
- **Problem:** Wszystkie moduły są importowane na sztywno w `bootstrap()`. Dodanie nowego modułu wymaga modyfikacji `main.py` — łamie zasadę otwarte/zamknięte.
- **Poprawka:** Zaimplementować auto-discovery:

```python
# _projects/_utils/module_discovery.py:
import importlib
import pkgutil
from pathlib import Path

def discover_modules(package: str, contract_type) -> list:
    """Automatycznie wykrywa moduły implementujące dany kontrakt."""
    package_root = importlib.import_module(package)
    modules = []
    for importer, modname, ispkg in pkgutil.walk_packages(
        package_root.__path__, 
        package_root.__name__ + "."
    ):
        if modname.endswith("_service") or "service" in modname.lower():
            module = importlib.import_module(modname)
            for attr_name in dir(module):
                attr = getattr(module, attr_name)
                if (isinstance(attr, type) and 
                    issubclass(attr, contract_type) and 
                    attr is not contract_type):
                    modules.append(attr())
    return modules
```

### 6. Godzina 12 — niejednoznaczna granica poranka/popołudnia
- **Lokalizacja:** `_projects/modules/polish_greeting.py` i `_projects/modules/english_greeting.py`
- **Problem:** Godzina 12 jest brana jako "rano" (6-12), ale 12:00 to technicznie południe — powinno należeć do "popołudnie". Granice przedziałów są niejednoznaczne.
- **Poprawka:** Użyć `6 ≤ hour < 12` dla rana i `12 ≤ hour < 18` dla popołudnia. Wyjaśnić w dokumentacji.

```python
# _projects/_utils/time_greeting_helper.py:
def get_greeting_period(hour: int) -> str:
    """Zwraca okres dnia na podstawie godziny.
    
    Granice:
        - noc:   0-5,  22-23
        - rano:  6-11
        - popołudnie: 12-17
        - wieczór: 18-21
    """
    if 6 <= hour < 12:
        return "morning"
    elif 12 <= hour < 18:
        return "afternoon"
    elif 18 <= hour < 22:
        return "evening"
    else:
        return "night"
```

### 7. SessionManager zapisuje JSON po KAŻDEJ rejestracji — problem wydajnościowy
- **Lokalizacja:** `_projects/registry/service_registry.py` (zakładając istnienie)
- **Problem:** Przy dużej liczbie modułów (setki) nadmiarowe I/O spowalnia bootstrap.
- **Poprawka:** Zastosować buforowanie:

```python
class ServiceRegistry:
    def __init__(self, auto_save: bool = False):
        self._auto_save = auto_save
        self._dirty = False
    
    def register(self, ...):
        # ... rejestracja ...
        self._dirty = True
        if self._auto_save:
            self._save_snapshot()
    
    def flush(self):
        """Zapisz wszystkie oczekujące zmiany."""
        if self._dirty:
            self._save_snapshot()
            self._dirty = False
```
